# Error Handling

The SecureLend SDK uses custom error classes that extend the built-in `Error` object. This makes it easy to handle different types of errors that may occur during an API call.

All SDK errors are instances of `SecureLendError`.

## Error Handling Example

Here's a comprehensive example of how to handle various error types using a `try...catch` block.

```typescript
import {
  SecureLendError,
  AuthenticationError,
  RateLimitError,
  ValidationError,
  NetworkError,
} from '@securelend/sdk';

try {
  // Example request that might fail
  const loans = await securelend.loans.compare({ amount: 1000 /* invalid */ });
} catch (error) {
  if (error instanceof AuthenticationError) {
    console.error('Authentication Error:', error.message);
    // Handle invalid API key
  } else if (error instanceof RateLimitError) {
    console.error('Rate Limit Error:', error.message);
    // Handle rate limiting (e.g., retry with backoff)
  } else if (error instanceof ValidationError) {
    console.error('Validation Error:', error.message);
    // Handle invalid input from the user
  } else if (error instanceof NetworkError) {
    console.error('Network Error:', error.message);
    // Handle network connectivity issues
  } else if (error instanceof SecureLendError) {
    console.error('SecureLend API Error:', error.message);
    // Handle generic API errors
  } else {
    console.error('An unexpected error occurred:', error.message);
  }
}
```

## Error Types

#### `SecureLendError`
The base error class for all SDK-specific errors. Catching this will catch any error originating from the SecureLend SDK.

#### `AuthenticationError`
Thrown when the API key is missing, invalid, or expired.

#### `RateLimitError`
Thrown when the API rate limit has been exceeded for your account.

#### `ValidationError`
Thrown when the request parameters are invalid (e.g., missing a required field, amount too low).

#### `NotFoundError`
Thrown when a requested resource could not be found.

#### `ServerError`
Thrown for internal errors on SecureLend's servers.

#### `NetworkError`
Thrown when there is a problem with the network connection during the request.
